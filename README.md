# Telegram Design News Bot

Дизайнерский новостной канал на автопилоте. Каждый день в 17:00 МСК берёт свежее из RSS, прогоняет через Claude (короткий русский пересказ + практическая польза + хэштеги), постит в канал с картинкой.

## Как это работает

**Расписание.** Один прогон в день, 17:00 МСК. На каждый день недели — свой набор рубрик, чтобы канал не зацикливался:

- ПН: Дизайн, Типографика
- ВТ: Архитектура, Веб и интерфейсы
- СР: Студии и блоги, Русский дизайн
- ЧТ: Искусство
- ПТ: Анимация, Кино
- СБ: Музыка, Дизайн
- ВС: Искусство, Типографика

За неделю прокатываются все 10 рубрик. Внутри дня бот берёт 3 самых свежих материала из источников этих рубрик и публикует с интервалом ~4 секунды.

**Пересказ.** Каждый материал прогоняется через Claude Haiku 4.5 со строгим промптом: живой русский язык, никакого канцелярита, не выдумывать факты сверх источника. На выходе — заголовок-тезис, короткий пересказ, блок с практической пользой, тематические хэштеги.

**Структура поста.**

```
🎨 Дизайн · Pentagram

[Заголовок-тезис на русском]

[Пересказ в 2-3 предложения]

💡 [Что отсюда можно вынести]

#дизайн #брендинг #айдентика

[ссылка на оригинал]
```

Картинка превью подтягивается из RSS, либо из og:image страницы как фоллбэк.

**Дедупликация.** Все отправленные ID хранятся в `sent_items.json`, файл коммитится обратно в репо после каждого прогона.

## Про «фактчекинг»

Сразу проговорю честно. Настоящий фактчекинг — это перекрёстная проверка по нескольким независимым источникам, и это отдельная большая задача. Что бот делает реально:

1. Промпт жёстко запрещает модели выдумывать имена, цифры и факты, которых нет в оригинале. Температура 0.3, ответ строго в JSON.
2. В каждом посте остаётся ссылка на первоисточник — для проверки руками.

Этого достаточно, чтобы пересказ не врал относительно оригинала. Но если оригинал неточен — бот это не поймает. Имей в виду.

## Запуск

### 1. Бот в BotFather

`/newbot`, сохранить токен.

### 2. Бот в админы канала

Минимум — права на отправку сообщений и медиа. ID канала: `@username` для публичного, числовой (с минусом) для приватного — узнаётся через [@userinfobot](https://t.me/userinfobot).

### 3. API-ключ Anthropic

[console.anthropic.com](https://console.anthropic.com) → API Keys → Create Key. Закинь $5 на баланс — этого хватит на пару лет работы бота при текущем темпе. Ориентировочная стоимость: ~$0.30 в месяц (3 поста в день на Haiku 4.5).

### 4. GitHub

```bash
git init
git add .
git commit -m "init"
gh repo create design-news-bot --public --source=. --push
```

Делай **публичный** репо — у публичных безлимитные минуты Actions.

### 5. Секреты репо

Settings → Secrets and variables → Actions → New repository secret:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `ANTHROPIC_API_KEY`

### 6. Проверь ленты

Локально, до первого боевого прогона:

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python bot.py check
```

Скрипт пройдёт по всем 50 источникам и в конце выдаст список мёртвых. Их нужно либо выкинуть из `SOURCES` в `bot.py`, либо подменить URL. Особенно подозрительны русские издания — у некоторых RSS закрыт.

### 7. Первый запуск

Actions → News bot → Run workflow. Первый прогон только инициализирует state, постов не будет. Со следующего пойдут материалы.

## Настройка

Всё в верху `bot.py`:

- `SOURCES` — добавляй и убирай источники.
- `DAILY_CATEGORIES` — расписание рубрик по дням недели. Можешь менять как хочешь.
- `POSTS_PER_DAY` — сколько постить за день. Сейчас 3.
- `MAX_AGE_HOURS` — игнорировать материалы старше N часов.
- `CLAUDE_MODEL` — модель для пересказа. По умолчанию Haiku 4.5 (дёшево и быстро). Если хочешь стиль покрасивее — `claude-sonnet-4-5`, но дороже в 5–10 раз.

Время — в `.github/workflows/post.yml`, строка `cron`. Сейчас `'0 14 * * *'` (14:00 UTC = 17:00 МСК). Если живёшь в другом поясе — пересчитай.

## Локальная отладка

```bash
export TELEGRAM_BOT_TOKEN="..."
export TELEGRAM_CHAT_ID="@your_channel"
export ANTHROPIC_API_KEY="sk-ant-..."
python bot.py        # боевой прогон
python bot.py check  # проверка лент
```

## Если что не так

- **Перевод плохой / странный.** Поправь промпт в константе `REWRITE_PROMPT` в `bot.py`. Это самое мощное место для тонкой настройки голоса канала.
- **Картинка не подтянулась.** Telegram сам подтягивает превью по ссылке, обычно нормально получается.
- **Лента отдаёт пусто.** `python bot.py check` покажет.
- **Хочется больше постов.** Подними `POSTS_PER_DAY` или добавь второй прогон в cron.
- **Хочется только русский / только английский.** Убери ненужные источники из `SOURCES`.
