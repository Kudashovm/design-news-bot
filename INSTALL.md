# Запуск бота с нуля

Подробная инструкция. Без терминала, без программирования — только браузер и Telegram. Времени всё займёт около 30 минут.

---

## Что понадобится

- Аккаунт в Telegram
- Аккаунт на GitHub (бесплатный, github.com)
- Аккаунт в Anthropic (console.anthropic.com) и банковская карта для пополнения на $5
- Распакованная папка `bot` с файлами бота (из архива, который я тебе скинул)

---

## Шаг 1. Создаём канал в Telegram

Если канал уже есть — переходи к Шагу 2.

1. В Telegram нажми меню (три полоски слева сверху на телефоне или иконку карандаша на десктопе)
2. **Создать канал** → имя, описание, аватар
3. Тип канала:
   - **Публичный** (с username вида `@my_design_news`) — проще всего, бот к нему легко подключается. Рекомендую этот вариант.
   - **Приватный** — тоже можно, но придётся возиться с числовым ID канала (расскажу на Шаге 4).
4. Готово, канал создан.

---

## Шаг 2. Создаём бота через BotFather

1. В поиске Telegram найди **@BotFather** (это официальный бот Telegram, у него галочка верификации).
2. Открой и нажми **Start**.
3. Отправь команду: `/newbot`
4. BotFather спросит **имя** бота — это то, что будет видно в шапке профиля. Например: `Дизайн новости`
5. Затем спросит **username** — должен заканчиваться на `bot`. Например: `design_news_max_bot` (если занято — придумай другой).
6. BotFather пришлёт сообщение с **токеном** — длинная строка вида:
   ```
   7891234567:AAH-XXxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
7. **Сохрани этот токен в надёжное место** (заметки, менеджер паролей). Он понадобится один раз. Никому его не показывай — кто его получит, сможет управлять ботом.

---

## Шаг 3. Добавляем бота в канал админом

1. Открой свой канал в Telegram.
2. Тапни на название канала вверху → **Управление каналом** (или Edit Channel).
3. **Администраторы** → **Добавить администратора**.
4. В поиске введи username своего бота (тот, что заканчивается на `bot`).
5. Выбери его, выдай права. Минимум, что нужно:
   - **Публикация сообщений** ✅
   - Остальное можно отключить.
6. Сохрани.

---

## Шаг 4. Узнаём ID канала

**Если канал публичный** — всё просто. ID — это `@username` твоего канала (тот же, что в ссылке `t.me/имя_канала`). Например: `@my_design_news`.

**Если канал приватный** — нужен числовой ID:

1. В Telegram найди бота **@userinfobot** или **@RawDataBot** (любой из них).
2. Открой свой канал, выбери любой пост (или просто отправь любое сообщение в канал, если ещё пусто).
3. Перешли это сообщение в @userinfobot.
4. Бот ответит, и в его ответе будет строка типа:
   ```
   Forwarded from chat: -1001234567890
   ```
5. **Этот номер с минусом и есть ID канала.** Сохрани его.

---

## Шаг 5. Получаем API-ключ Anthropic

Это нужно, чтобы Claude переводил и пересказывал новости.

1. Открой [console.anthropic.com](https://console.anthropic.com), нажми **Sign Up**.
2. Зарегистрируйся (email + пароль или Google). Возможна верификация телефона.
3. После входа: левое меню → **Plans & Billing** → **Add to balance** или **Buy credits**.
4. Пополни баланс на минимум **$5** (привязать карту). Этого хватит примерно на год работы бота при текущем темпе (3 поста в день).
5. Левое меню → **API Keys** → **Create Key**.
6. Имя ключа: `design-bot` (любое).
7. Скопируй ключ (`sk-ant-...`). **Сохрани в надёжное место — после закрытия окна его уже не покажут.**

---

## Шаг 6. Регистрируемся на GitHub

Если аккаунт уже есть — следующий шаг.

1. Открой [github.com](https://github.com) → **Sign up**.
2. Email, пароль, username, верификация.
3. Бесплатный план (Free) — этого достаточно.

---

## Шаг 7. Создаём репозиторий

1. На github.com нажми зелёную кнопку **+** (вверху справа) → **New repository**.
2. **Repository name**: `design-news-bot` (или любое латиницей без пробелов).
3. **Public** ✅ — обязательно публичный, потому что у публичных репозиториев безлимитные минуты GitHub Actions. Не переживай: код безобидный, секреты в нём не лежат.
4. Никаких галочек: **не** добавлять README, gitignore, лицензию.
5. **Create repository**.

---

## Шаг 8. Заливаем файлы

Самый простой способ — через веб-интерфейс GitHub.

1. На странице свежего репозитория ты увидишь подсказку: **uploading an existing file** (или ссылку **upload files**). Нажми её.
2. Открой на компьютере распакованную папку `bot` из архива.
3. **Перетащи в окно браузера всё содержимое папки** `bot/`:
   - `bot.py`
   - `requirements.txt`
   - `README.md`
   - `INSTALL.md` (этот файл)
   - `sent_items.json`
   - `.gitignore`
   - папку `.github` целиком (с подпапкой `workflows` и файлами `post.yml`, `check.yml` внутри)

   Важно: перетаскивай **содержимое** папки `bot/`, а не саму папку. То есть открой папку `bot/`, выдели всё внутри (Cmd+A на маке) и тяни.

4. Внизу страницы — **Commit changes**. Нажми.

После этого в репозитории должна появиться структура:
```
bot.py
requirements.txt
README.md
INSTALL.md
sent_items.json
.gitignore
.github/
  workflows/
    post.yml
    check.yml
```

Проверь, что папка `.github/workflows` залилась — это критично. Если её не видно — попробуй залить ещё раз или используй [GitHub Desktop](https://desktop.github.com/) (есть для Mac и Windows, перетаскивает папки надёжнее).

---

## Шаг 9. Разрешаем GitHub Actions писать в репозиторий

Это критичный шаг — без него бот не сможет запоминать, что уже отправлено, и будет дублировать посты.

1. В репозитории: **Settings** (вкладка наверху).
2. Левое меню → **Actions** → **General**.
3. Прокрутить до раздела **Workflow permissions**.
4. Выбрать **Read and write permissions** ✅
5. **Save**.

---

## Шаг 10. Прописываем секреты

Здесь сохранятся токены, чтобы они были доступны для бота, но не лежали в коде.

1. **Settings** → левое меню → **Secrets and variables** → **Actions**.
2. **New repository secret**.
3. Создай **три** секрета (имена должны быть точно такие, как ниже):

   | Имя | Значение |
   |---|---|
   | `TELEGRAM_BOT_TOKEN` | Токен бота от BotFather (Шаг 2) |
   | `TELEGRAM_CHAT_ID` | `@username_канала` или числовой ID (Шаг 4) |
   | `ANTHROPIC_API_KEY` | Ключ от Anthropic (Шаг 5) |

После создания каждого секрета на странице будут видны только имена — это нормально, значения скрыты в целях безопасности.

---

## Шаг 11. Проверяем, что ленты живые

Перед боевым запуском полезно проверить, какие RSS-источники работают.

1. Вкладка **Actions** в репозитории.
2. В левом списке выбери **Check feeds**.
3. Справа — кнопка **Run workflow** → **Run workflow** (зелёная).
4. Через 5-10 секунд появится новый запуск. Кликни на него, потом на джоб **check**.
5. Открой шаг **Check all feeds** — там будет список всех источников с галочками или крестиками.

Если в конце есть строки вида:
```
Мёртвые — выкинь из SOURCES в bot.py или замени URL:
  Дизайн · Communication Arts → ...
  Искусство · The Blueprint → ...
```

— значит эти ленты не отвечают. Их можно убрать:

1. В репозитории открой `bot.py`.
2. Кнопка карандаша (Edit) справа сверху.
3. Найди строки с проблемными источниками в словаре `SOURCES` и удали их (или закомментируй, поставив `#` в начале строки).
4. Внизу — **Commit changes**.

Можно сделать это сейчас или позже. Бот не падает на мёртвых лентах, просто пропускает их.

---

## Шаг 12. Первый запуск (без постов)

Первый прогон бота не публикует ничего, только запоминает, какие материалы уже есть в лентах. Это чтобы канал не завалило сотней старых статей сразу.

1. **Actions** → **News bot** → **Run workflow** → **Run workflow**.
2. Подождать 1-3 минуты.
3. Открыть запуск, открыть джоб **post**, открыть шаг **Run bot**. В логах должно быть что-то вроде:
   ```
   Сегодня Friday 09.05, рубрики: Анимация, Кино
   Первый запуск: записано N ID, без постинга.
   ```
4. После завершения шага **Commit state** в коде появится коммит `chore: update sent items` — значит, файл состояния обновился.

---

## Шаг 13. Тестовый боевой пост

1. Подожди минут 30-60, чтобы в лентах появились свежие материалы (на момент первого запуска все материалы записаны как «уже отправленные», поэтому сразу постить нечего).
2. **Actions** → **News bot** → **Run workflow**.
3. Через пару минут проверь канал — там должен появиться пост (или несколько, если есть свежее).

Если поста нет — открой логи запуска. Скорее всего в логах: `Новых записей нет.` Это значит, что свежее ещё не вышло. Подожди и попробуй позже.

---

## Шаг 14. Расписание

Дальше всё работает само. Бот автоматически запускается каждый день в **17:00 МСК** (14:00 UTC).

Учти: GitHub Actions cron не точен до минуты, в реальности запуск может произойти в 17:05–17:15. Это нормально.

**Важный нюанс:** если в репозиторий 60 дней не было активности, GitHub автоматически отключит расписание (это политика для экономии ресурсов). Достаточно сделать любой коммит раз в пару месяцев — например, поправить какой-нибудь источник. Каждый автокоммит state-файла тоже считается активностью, так что пока бот работает — всё ок.

## Что менять, если захочешь

Все настройки — в начале файла `bot.py`. Открой через карандаш в веб-интерфейсе.

- **Источники** — словарь `SOURCES`. Добавляй пары `(имя, url)` в нужную рубрику.
- **Расписание рубрик по дням** — `DAILY_CATEGORIES`. Сейчас:
  - ПН: Дизайн + Типографика
  - ВТ: Архитектура + Веб
  - СР: Студии и блоги + Русский дизайн
  - ЧТ: Искусство
  - ПТ: Анимация + Кино
  - СБ: Музыка + Дизайн
  - ВС: Искусство + Типографика
- **Сколько постить за день** — `POSTS_PER_DAY` (сейчас 3).
- **Стиль текста** — самое мощное место — `REWRITE_PROMPT`. Это инструкция для Claude, как перепаковывать материал. Меняй фразы, добавляй свои правила, запрещай слова.
- **Время** — в `.github/workflows/post.yml`, строка `cron`. Сейчас `'0 14 * * *'` (это 14:00 UTC = 17:00 МСК). Если хочешь другое время:
  - Утро (9:00 МСК): `'0 6 * * *'`
  - Полдень (12:00 МСК): `'0 9 * * *'`
  - Вечер (20:00 МСК): `'0 17 * * *'`

После любого изменения — **Commit changes** внизу страницы. Workflow подхватит новые настройки в следующий запуск.

## Если что-то сломалось

**Посты не приходят в канал.**
- Проверь, что бот реально админ канала с правом постинга (Шаг 3).
- Проверь, что `TELEGRAM_CHAT_ID` записан правильно. Для приватного — с минусом, например `-1001234567890`.
- Открой логи последнего запуска в Actions — там будет конкретная ошибка.

**Workflow падает с ошибкой про секрет.**
- Перепроверь имена секретов в Settings → Secrets. Они должны быть **точно** `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `ANTHROPIC_API_KEY` (с подчёркиваниями, заглавными).

**Бот постит дубли.**
- Скорее всего пропущен Шаг 9 (разрешения на запись). Без них state-файл не коммитится, и бот не помнит, что уже отправлял.

**Пост уродский / не нравится стиль.**
- Открой `bot.py`, найди константу `REWRITE_PROMPT`. Это промпт для Claude. Можешь переписать его как угодно — добавить примеры хорошего и плохого, запретить какие-то обороты, попросить начинать с цифры. Это самое мощное место для настройки голоса.

**Anthropic API возвращает ошибку «no credit».**
- Пополни баланс на console.anthropic.com → Plans & Billing.

**Хочется протестировать без публикации.**
- Запусти **Check feeds** workflow — он только проверяет источники, в канал ничего не шлёт.
